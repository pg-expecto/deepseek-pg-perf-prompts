# deepseek-pg-perf-prompts
Коллекция промптов для нейросети **DeepSeek** (на русском языке), предназначенных для анализа и оптимизации производительности СУБД **PostgreSQL**.


---

## 📦 Требования

- PostgreSQL 15+ (проверялось на версиях 15 и выше)
- Аккаунт DeepSeek (бесплатного тарифа достаточно для большинства промптов)
- Базовое знание SQL для подстановки своих значений

## 📘 Инструкция для анализа производительности

В репозитории присутствует файл **`_prompt-instruction.txt`** – это детальная инструкция (промпт) для нейросети DeepSeek, задающая роль эксперта по производительности PostgreSQL и строгие правила анализа.

### Что содержит инструкция?

- Роль: эксперт по производительности СУБД PostgreSQL.
- Правила ответов: опора только на предоставленные данные, указание недостающих метрик, профессиональная терминология (shared_buffers, seq scan, autovacuum и т.д.).
- Уровни достоверности выводов (от «подтверждено данными» до «невозможно оценить»).
- Протокол обработки противоречий в данных.
- Требования к стилю и границы применимости.

### Как использовать инструкцию?

1. Откройте диалог с DeepSeek (чат).
2. Скопируйте полное содержимое файла `_prompt-instruction.txt` и отправьте его первым сообщением.
3. Затем присылайте свои данные (логи, метрики, конфигурацию) и вопросы по производительности PostgreSQL.

**Важно:** Инструкция заставляет DeepSeek быть строгим, предметным и не додумывать факты. Для получения качественного анализа обязательно предоставляйте как можно больше конкретных метрик (время наблюдения, настройки PostgreSQL, выдержки из pg_stat_* и т.п.).

### Пример использования

> Вы (первое сообщение):  
> *[вставлено содержимое _prompt-instruction.txt]*  
>  
> Вы (второе сообщение):  
> «Период наблюдения: 2025-02-10 14:00–15:00.  
> shared_buffers = 2GB, work_mem=4MB, checkpoint_timeout=5min.  
> hit ratio = 98%, но среднее время ответа запросов выросло с 10ms до 50ms.  
> Топ ожиданий: DataFileRead – 65%, WALWrite – 20%...»

DeepSeek проанализирует данные, укажет уровни достоверности и запросит недостающие метрики, если потребуется.

## 🤝 Внесение вклада

Мы приветствуем новые промпты и улучшения!

1. Сделайте форк репозитория.
2. Добавьте свой промпт в папку `prompts/` (или прямо в README как пример).
3. Убедитесь, что промпт достаточно **общий** (используйте плейсхолдеры) и при этом **специфичен для производительности PostgreSQL**.
4. Откройте Pull Request.

Пожалуйста, кратко опишите, какую проблему производительности решает ваш промпт.
