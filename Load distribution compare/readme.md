# Анализ конфигурационных параметров СУБД и распределения нагрузки (pgpro_pwr)

## Назначение

Данный README описывает последовательность действий для подготовки аналитического отчёта о влиянии изменений конфигурационных параметров СУБД на показатели «Load distribution» на основе отчётов pgpro_pwr.

## Необходимые файлы и инструкции

Перед началом работы убедитесь, что у вас есть следующие файлы:

- `_prompt-instruction.txt` – основные инструкции для анализа (доступен в [репозитории deepseek-pg-perf-prompts](https://github.com/pg-expecto/deepseek-pg-perf-prompts/tree/main))
- [Philosophical_instruction_BETA_v5.1.md](https://github.com/pg-expecto/Philosophical_instruction/blob/main/Philosophical_instruction_BETA_v5.1.md) (методологические указания)
- `load_distribution.1.txt` – таблица «Load distribution» по первому отчёту pgpro_pwr (например, отчёт-X1-X2)
- `load_distribution.2.txt` – таблица «Load distribution» по второму отчёту pgpro_pwr (например, отчёт-Y1-Y2)
- `Таблица_G.78.txt` – описание раздела «Cluster settings during the report interval»
- `ТаблицаG.25.txt` – описание метрик (`Total time`, `Executed count`, `I/O time` и др.)
- `table.result.txt` – таблица соответствия конфигурационных параметров СУБД между отчётами, сформированная с помощью [инструмента сравнения](https://github.com/pg-expecto/deepseek-pg-perf-prompts/tree/main/parameter%20table)
- `prompt-analyze.txt` – шаблон аналитического запроса (требует ручной правки)

## Последовательность действий

Выполните шаги в указанном порядке:

1. **Создайте новый чат** с DeepSeek.

2. **Загрузите инструкции** в чат:
   - `_prompt-instruction.txt` (скачайте его из репозитория [pg-expecto/deepseek-pg-perf-prompts](https://github.com/pg-expecto/deepseek-pg-perf-prompts/tree/main))
   - `Philosophical_instruction_BETA_v5.1.md` (можно по ссылке или локально)

3. **Введите первый запрос** DeepSeek:
   ```
   Строго следуй инструкции, ожидай данные
   ```

4. **Загрузите данные** (последовательная отправка файлов):
   - `load_distribution.1.txt` – таблица нагрузки по первому отчёту
   - `load_distribution.2.txt` – таблица нагрузки по второму отчёту
   - `Таблица_G.78.txt` – описание параметров кластера
   - `ТаблицаG.25.txt` – описание метрик распределения нагрузки
   - `table.result.txt` – таблица сравнения параметров

5. **Отредактируйте файл `prompt-analyze.txt`**:
   - Замените `X1-X2` на фактические идентификаторы из названия **первого** отчёта pgpro_pwr.
   - Замените `Y1-Y2` на фактические идентификаторы из названия **второго** отчёта pgpro_pwr.

6. **Скопируйте содержимое исправленного `prompt-analyze.txt`** и отправьте его DeepSeek как второй запрос.

7. **Выполните запрос** – дождитесь завершения генерации ответа.

8. **Сохраните полученный результат** в файл `table.source.1.txt`.

## Примечания

- В ответе вместо цветовой маркировки инструкции `Philosophical_instruction_BETA_v5.1.md` используются текстовые замены:
  - `"Подтверждено"`
  - `"Вероятно"`
  - `"Предположение"`
  - `"Неизвестно"`
- Названия отчётов (`отчёт-X1-X2`, `отчёт-Y1-Y2`) берите из реальных имён ваших файлов с отчётами pgpro_pwr.
- Итоговый аналитический отчёт (результат выполнения запроса) будет содержать:
  - ключевые отличия конфигурационных параметров;
  - ключевые отличия показателей «Load distribution»;
  - гипотезу о влиянии конфигурации на нагрузку;
  - последовательность рассуждений и способы проверки гипотезы.

## Дополнительная информация

- Инструмент для формирования `table.result.txt`: [deepseek-pg-perf-prompts/parameter table](https://github.com/pg-expecto/deepseek-pg-perf-prompts/tree/main/parameter%20table)
- При возникновении вопросов сверяйтесь с исходным `prompt-analyze.txt` и описаниями таблиц G.78 и G.25.
