# Сводный анализ влияния изменений на профиль распределения нагрузки СУБД 

## Назначение

Данный README описывает последовательность действий для подготовки аналитического отчёта о влиянии изменений конфигурационных параметров СУБД на показатели «Load distribution» на основе отчётов pgpro_pwr по результатам сравнения двух отчетов.

## Необходимые файлы и инструкции

Перед началом работы убедитесь, что у вас есть следующие файлы:

- `_prompt-instruction.txt` – доменная инструкция для анализа (доступна в [репозитории deepseek-pg-perf-prompts](https://github.com/pg-expecto/deepseek-pg-perf-prompts/tree/main))
- `Philosophical_instruction_BETA_v5.1.md` - философская инструкция  для анализа (доступна в [репозитории Philosophical_instruction_BETA_v5.1.md](https://github.com/pg-expecto/Philosophical_instruction/blob/main/Philosophical_instruction_BETA_v5.1.md))
- ТаблицаG.25.txt : описание раздела "Load distribution (Распределение нагрузки)"
- ТаблицаG.33.txt : описание раздела "Top SQL by execution time (SQL-запросы с наибольшей длительностью выполнения)"
- ТаблицаG.36.txt : описание раздела "Top SQL by I/O wait time (SQL-запросы с наибольшим временем I/O)"
- prompt-load_dist-X1-X2.txt - промпт для подготовки исходных данных по профилу нагрузки СУБД по первому отчету  (требует ручной правки)
- prompt-load_dist-Y1-Y2.txt - промпт для подготовки исходных данных по профилу нагрузки СУБД по второму отчету  (требует ручной правки)
- prompt-top-sql-1-X1-X2.txt - промпт для подготовки исходных данных "Top SQL by execution time" по первому отчету  (требует ручной правки)
- prompt-top-sql-1-Y1-Y2.txt - промпт для подготовки исходных данных "Top SQL by execution time" по второму отчету  (требует ручной правки)
- prompt-top-sql-2-X1-X2.txt - промпт для подготовки исходных данных "Top SQL by I/O wait time" по первому отчету  (требует ручной правки)
- prompt-top-sql-2-Y1-Y2.txt - промпт для подготовки исходных данных "Top SQL by I/O wait time" по второму отчету  (требует ручной правки)
- prompt_final_analyze.txt - промпт для подготовки сводного аналитического отчета (требует ручной правки)



## Последовательность действий

Выполните шаги в указанном порядке:

### Формирование исходных данных по профилю нагрузки

1. **Создайте новый чат** с DeepSeek.

2. **Загрузите инструкции** в чат:
   - `_prompt-instruction.txt` 

3. **Введите первый запрос** DeepSeek:
   ```
   Строго следуй инструкции, ожидай данные
   ```

4. **Отредактируйте файл `prompt-load_dist-X1-X2.txt`**:
   - Замените `X1-X2` на фактические идентификаторы из названия **первого** отчёта pgpro_pwr.

5. **Скопируйте содержимое исправленного `prompt-load_dist-X1-X2.txt`** и отправьте его DeepSeek как второй запрос.

6. **Выполните запрос** – дождитесь завершения генерации ответа.

6. **Сохраните полученный результат** в файл `load_dist-X1-X2.txt`. Замените `X1-X2` на фактические идентификаторы из названия **первого** отчёта pgpro_pwr.

7. **Отредактируйте файл `prompt-load_dist-Y1-Y2.txt`**:
   - Замените `Y1-Y2` на фактические идентификаторы из названия **первого** отчёта pgpro_pwr.

9. **Скопируйте содержимое исправленного `prompt-load_dist-X1-X2.txt`** и отправьте его DeepSeek как второй запрос.

10. **Выполните запрос** – дождитесь завершения генерации ответа.

11. **Сохраните полученный результат** в файл `load_dist-Y1-Y2.txt`. Замените `Y1-Y2` на фактические идентификаторы из названия **второго** отчёта pgpro_pwr.

### Формирование **Сводного аналитического отчета**

1. **Создайте новый чат** с DeepSeek.

2. **Загрузите инструкции** в чат:
   - `_prompt-instruction.txt` 
   - `Philosophical_instruction_BETA_v5.1.md` 

3. **Введите первый запрос** DeepSeek:
   ```
   Строго следуй инструкции, ожидай данные
   ```

4. **Отредактируйте файл `prompt_final_analyze.txt`**:
   - Замените `X1-X2` на фактические идентификаторы из названия **первого** отчёта pgpro_pwr.
   - Замените `Y1-Y2` на фактические идентификаторы из названия **второго** отчёта pgpro_pwr.
   - Заменить `ОПИСАНИЕ ИЗМЕНЕНИЯ` на подробное описание проведенных изменений перед выполнением **второго** отчета
   - Заменить `Ссылка на документацию по изменению` на ссылки на документацию по измененным параметрам

5. **Скопируйте содержимое исправленного `prompt_final_analyze.txt`** и отправьте его DeepSeek как второй запрос.

7. **Выполните запрос** – дождитесь завершения генерации ответа.

8. **Сохраните полученный результат** в файл `final-analyze.txt`.

## Примечания

- В ответе вместо цветовой маркировки инструкции `Philosophical_instruction_BETA_v5.1.md` используются текстовые замены:
  - `"Подтверждено"`
  - `"Вероятно"`
  - `"Предположение"`
  - `"Неизвестно"`
- Итоговый аналитический отчёт (результат выполнения запроса) будет содержать:
  - ключевые отличия показателей «Load distribution»;
  - гипотезу о влиянии конфигурации на нагрузку;
  - последовательность рассуждений и способы проверки гипотезы.
