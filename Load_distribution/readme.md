# README — описание вложенных файлов

## Файлы

### 1. `_prompt-instruction.txt`
Инструкция для ИИ-эксперта по производительности PostgreSQL. Задаёт правила анализа метрик, уровни достоверности выводов (от L1 до L4), протокол обработки противоречий и требования к стилю ответов. Используется как системный промпт при анализе данных о работе СУБД.

### 2. `prompt-load.source.txt`
Техническое задание на извлечение данных из HTML-отчёта `pgpro_pwr.X1-X2.clear.html`. Предписывает:
- Извлечь массив `_data[0].report_data.datasets.load_dist`.
- Отфильтровать объекты с `db_filter: true` (строки баз данных).
- Сопоставить метрики из Таблицы G.25 с полями JSON (например, `Total time` → `total_time`).
- Построить markdown-таблицу абсолютных значений для каждой базы данных, используя поле `objname` как название БД.
- Формат таблицы: строки — метрики, столбцы — базы данных, числовые значения выравнены вправо.

### 3. `ТаблицаG.25.txt`
Описание раздела «Load distribution» отчёта `pgpro_pwr`. Содержит список метрик (Total time, Executed count, I/O time, Blocks fetched, Shared blocks read, Shared blocks dirtied, Shared blocks written, WAL generated, Temp and Local blocks written, Temp and Local blocks read, Invalidation messages sent, Cache resets) с пояснениями и формулами расчёта. Используется в `prompt-load.source.txt` для определения набора строк итоговой таблицы.

## Взаимосвязь файлов
- `ТаблицаG.25.txt` задаёт словарь метрик.
- `prompt-load.source.txt` использует этот словарь для маппинга на поля JSON и определяет структуру выходной таблицы.
- `_prompt-instruction.txt` не используется непосредственно в задании из `prompt-load.source.txt`, но может применяться при дальнейшем анализе полученных данных (например, для интерпретации значений метрик).
