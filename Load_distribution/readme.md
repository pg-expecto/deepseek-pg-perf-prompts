## Файлы проекта

### 1. `_prompt-instruction.txt`
Инструкция для ИИ-эксперта по производительности PostgreSQL. Задаёт правила анализа метрик, уровни достоверности выводов (L1–L4), протокол обработки противоречий и требования к стилю ответов. Используется как системный промпт при анализе данных о работе СУБД.

### 2. `prompt-load.source.txt`
Техническое задание на извлечение данных из HTML-отчёта `pgpro_pwr.X1-X2.clear.html`. Предписывает:
- Извлечь массив `_data[0].report_data.datasets.load_dist`.
- Отфильтровать объекты с `db_filter: true` (строки баз данных).
- Сопоставить метрики из Таблицы G.25 с полями JSON.
- Построить markdown-таблицу абсолютных значений для каждой базы данных, используя поле `objname` как название БД.
- Формат таблицы: строки — метрики, столбцы — базы данных, числовые значения выравнены вправо.

### 3. `ТаблицаG.25.txt`
Описание раздела «Load distribution» отчёта `pgpro_pwr`. Содержит список метрик с пояснениями и формулами расчёта:
- Total time (sec.)
- Executed count
- I/O time (sec.)
- Blocks fetched
- Shared blocks read
- Shared blocks dirtied
- Shared blocks written
- WAL generated (bytes)
- Temp and Local blocks written
- Temp and Local blocks read
- Invalidation messages sent
- Cache resets

Используется в `prompt-load.source.txt` для определения набора строк итоговой таблицы.

## Инструкция по получению таблицы распределения нагрузки

1. **Загрузить инструкцию для эксперта**  
   - Откройте новый чат с DeepSeek.  
   - Загрузите файл [`_prompt-instruction.txt`].  
   - Введите запрос:  
     ```
     Строго следуй инструкции, ожидай данные
     ```

2. **Подготовить файл с запросом на извлечение данных**  
   - Отредактируйте файл [`prompt-load.source.txt`]:  
     - Замените `X1-X2` на фактические значения из имени вашего отчёта.  
     - Пример: если отчёт называется `pgpro_pwr.2025-04-01.clear.html`, замените на `2025-04-01`.

3. **Загрузить анализируемый отчёт и таблицу описания метрик**  
   - Загрузите в чат файл отчёта `pgpro_pwr.<период>.clear.html`.  
   - Загрузите файл [`ТаблицаG.25.txt`].

4. **Выполнить запрос**  
   - Скопируйте содержимое исправленного файла [`prompt-load.source.txt`] и отправьте его как сообщение в чат.

5. **Сохранить результат**  
   - Результат (markdown-таблица) сохраните в файл `load_distribution.1.txt` или `load_distribution.2.txt` (в зависимости от периода).

## Взаимосвязь файлов
- `ТаблицаG.25.txt` задаёт словарь метрик.  
- `prompt-load.source.txt` использует этот словарь для маппинга на поля JSON и определяет структуру выходной таблицы.  
- `_prompt-instruction.txt` не используется непосредственно в задании из `prompt-load.source.txt`, но может применяться при дальнейшем анализе полученных данных (например, для интерпретации значений метрик).

## Примечания
- Все файлы должны быть в кодировке UTF-8.  
- В отчёте учитываются только базы данных с `db_filter: true`.  
- Поле `cache_resets` может быть `null` — в таблицу переносится как есть.
