# Сравнение параметров СУБД между отчетами pgpro_pwr

Данная инструкция описывает процесс формирования сравнительной таблицы параметров кластера PostgreSQL на основе двух отчетов `pgpro_pwr` (например, за разные периоды или с разных серверов).

## Этап 1 – Извлечение параметров из первого отчета

1. Загрузите инструкцию `_prompt-instruction.txt` в новый чат DeepSeek.
2. Введите запрос: `Строго следуй инструкции, ожидай данные`.
3. Отредактируйте файл `prompt-correspondence_table.source.1.txt`:
   - Замените плейсхолдер `X1-X2` на фактические идентификаторы снимков (например, `01-02` или `A-B`) из имени первого отчета `pgpro_pwr`.
4. Загрузите в чат:
   - первый отчет `pgpro_pwr` (файл `pgpro_pwr.X1-X2.clear.html`)
   - файл `Таблица.G.78.txt`
5. Отправьте исправленный файл `prompt-correspondence_table.source.1.txt` как запрос.
6. Сохраните полученный результат в файл `table.source.1.txt`.

## Этап 2 – Извлечение параметров из второго отчета

1. Загрузите инструкцию `_prompt-instruction.txt` в **новый** чат DeepSeek.
2. Введите запрос: `Строго следуй инструкции, ожидай данные`.
3. Отредактируйте файл `prompt-correspondence_table.source.2.txt`:
   - Замените плейсхолдер `Y1-Y2` на фактические идентификаторы снимков из имени второго отчета `pgpro_pwr`.
4. Загрузите в чат:
   - второй отчет `pgpro_pwr` (файл `pgpro_pwr.Y1-Y2.clear.html`)
   - файл `Таблица.G.78.txt`
5. Отправьте исправленный файл `prompt-correspondence_table.source.2.txt` как запрос.
6. Сохраните полученный результат в файл `table.source.2.txt`.

## Этап 3 – Формирование итоговой сравнительной таблицы

1. Загрузите инструкцию `_prompt-instruction.txt` в **новый** чат DeepSeek.
2. Введите запрос: `Строго следуй инструкции, ожидай данные`.
3. Отредактируйте файл `prompt-correspondence_table.result.txt`:
   - Замените `X1-X2` на идентификаторы снимков первого отчета (из этапа 1).
   - Замените `Y1-Y2` на идентификаторы снимков второго отчета (из этапа 2).
4. Загрузите в чат результаты предыдущих этапов:
   - `table.source.1.txt`
   - `table.source.2.txt`
5. Отправьте исправленный файл `prompt-correspondence_table.result.txt` как запрос.
6. Сохраните итоговый результат в файл `table.result.txt`.

## Примечания

- Все промежуточные файлы (`table.source.1.txt`, `table.source.2.txt`) сохраняются локально.
- Итоговая таблица `table.result.txt` содержит строки параметров (Setting), для которых значения в двух отчетах различаются или отсутствуют в одном из них.
- Формат итоговой таблицы: `Setting | отчет-X1-X2 | отчет-Y1-Y2`.
