# Сравнительный анализ конфигурации, нагрузки и планов выполнения по отчётам pgpro_pwr

Данный документ описывает последовательность из пяти этапов для получения аналитического отчёта, объединяющего:
- сравнение конфигурационных параметров кластера,
- сравнение распределения нагрузки (Load distribution),
- сопоставление SQL-запросов по тексту,
- извлечение и сравнение планов выполнения,
- итоговый анализ с использованием философской инструкции.

## Необходимые файлы и инструкции

Перед началом работы убедитесь, что у вас есть:

- `_prompt-instruction.txt` – [основная инструкция для DeepSeek](https://github.com/pg-expecto/deepseek-pg-perf-prompts/blob/main/_prompt-instruction.txt)
- `Philosophical_instruction_BETA_v5.1.md` – [методологическая инструкция](https://github.com/pg-expecto/Philosophical_instruction/blob/main/Philosophical_instruction_BETA_v5.1.md)
- Два отчёта `pgpro_pwr` в формате HTML:  
  - `pgpro_pwr.X1-X2.clear.html` (первый отчёт, обозначается `отчёт-X1-X2`)  
  - `pgpro_pwr.Y1-Y2.clear.html` (второй отчёт, обозначается `отчёт-Y1-Y2`)
- Файлы запросов (промптов):  
  `prompt-1.txt`, `prompt-2.txt`, `prompt-3.txt`, `prompt-4.txt`, `prompt-analyze.txt`
- Вспомогательные таблицы:  
  `Таблица_G.78.txt`, `ТаблицаG.25.txt`

## Этап 1 – Извлечение списка SQL-запросов из второго отчёта (`отчёт-Y1-Y2`)

1. Откройте **новый чат** с DeepSeek.
2. Загрузите инструкцию `_prompt-instruction.txt`.
3. Введите запрос:
   ```
   Строго следуй инструкции, ожидай данные
   ```
4. Отредактируйте файл `prompt-1.txt`:
   - Замените `Y1-Y2` на фактические идентификаторы снимков **второго** отчёта `pgpro_pwr`.
   - В этом же файле указан раздел `TOP-SQL` – для извлечения списка «Top SQL».
5. Загрузите исправленный `prompt-1.txt` как запрос в чат.
6. Выполните запрос.
7. Сохраните полученный результат в файл **`queryid-1.txt`**.

## Этап 2 – Сопоставление Query ID между отчётами по тексту SQL

1. Откройте **новый чат** с DeepSeek.
2. Загрузите инструкцию `_prompt-instruction.txt`.
3. Введите запрос:
   ```
   Строго следуй инструкции, ожидай данные
   ```
4. Загрузите в чат файл `queryid-1.txt` (получен на Этапе 1).
5. Отредактируйте файл `prompt-2.txt`:
   - Замените `X1-X2` на идентификаторы **первого** отчёта.
   - Замените `Y1-Y2` на идентификаторы **второго** отчёта.
6. Загрузите исправленный `prompt-2.txt` как запрос.
7. Выполните запрос.
8. Сохраните результат в файл **`queryid-2.txt`**.

7. Выполните запрос.
8. Сохраните результат в файл **`plan-3.txt`**.

## Этап 5 – Формирование итогового аналитического отчёта с анализом планов

1. Откройте **новый чат** с DeepSeek.
2. Загрузите инструкции:
   - `_prompt-instruction.txt`
   - `Philosophical_instruction_BETA_v5.1.md` (можно по ссылке или локально)
3. Введите запрос:
   ```
   Строго следуй инструкциям, ожидай данные
   ```
4. Последовательно загрузите следующие файлы:
   - `table.result.txt` – таблица сравнения конфигурационных параметров (формируется [инструментом сравнения параметров](https://github.com/pg-expecto/deepseek-pg-perf-prompts/tree/main/parameter%20table))
   - `analyze.txt` – аналитический отчёт о сравнении конфигурации и нагрузки (формируется [инструментом сравнения Load distribution](https://github.com/pg-expecto/deepseek-pg-perf-prompts/tree/main/Load%20distribution%20compare))
   - `plan-3.txt` (с Этапа 3)
   - `plan-4.txt` (с Этапа 4)
   - `queryid-2.txt` (с Этапа 2)
   - `Таблица_G.78.txt`
   - `ТаблицаG.25.txt`
5. Отредактируйте файл `prompt-analyze.txt`:
   - Замените `X1-X2` на идентификаторы первого отчёта.
   - Замените `Y1-Y2` на идентификаторы второго отчёта.
6. Загрузите исправленный `prompt-analyze.txt` как запрос в чат.
7. Выполните запрос.
8. Сохраните полученный результат в файл **`analyze-Top_SQL.txt`**.

## Примечания

- На всех этапах после загрузки инструкции `_prompt-instruction.txt` обязательно вводите стартовую фразу:  
  `Строго следуй инструкции, ожидай данные`
- В `prompt-analyze.txt` цветовые обозначения из `Philosophical_instruction_BETA_v5.1.md` автоматически заменяются на текстовые метки:  
  `"Подтверждено"`, `"Вероятно"`, `"Предположение"`, `"Неизвестно"`.
- Если на Этапе 5 какой-либо из входных файлов отсутствует, вернитесь к соответствующему этапу и сформируйте его заново.
- Все промежуточные файлы рекомендуется сохранять в локальной папке проекта.
