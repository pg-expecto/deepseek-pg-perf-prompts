Входные данные:  
- pgpro_pwr_diff.X1-X2-Y1-Y2.clear – встроенный JSON-массив load_dist внутри тега <script>.  
- ТаблицаG.25.txt – описание метрик (Total time, Executed count, I/O time и т.д.)  

Задача: 
Сравнительный анализ раздела "Load distribution among heavily loaded databases"
Извлеки из файла массив `_data[0].report_data.datasets.load_dist` и выбери объекты, у которых `db_filter: true` (это строки по базам данных ).  
"Interval 1" - соответствует отчету "X1-X2"
"Interval 2" - соответствует отчету "Y1-Y2"
Для каждой метрики из Таблицы G.25 найди соответствующее поле в этих объектах (см. сопоставление ниже) и построй таблицу абсолютных значений в формате mark down:

**Отчет-X1-X2-Y1-Y2**
**Resource** | **X1-X2-наименование базы данных-1** |..| **X1-X2-наименование базы данных-N** |**Y1-Y2-наименование базы данных-1** |..| **Y1-Y2-наименование базы данных-N** |

Где строки – метрики из Таблицы G.25 в том же порядке.

**Сопоставление метрик с полями JSON:**
- Total time (sec.) → total_time (число, секунды)
- Executed count → exec_count
- I/O time (sec.) → blks_io_time
- Blocks fetched → blks_fetched
- Shared blocks read → shared_blks_read
- Shared blocks dirtied → shared_blks_dirtied
- Shared blocks written → shared_blks_written
- WAL generated → wal_bytes
- Temp and Local blocks written → temp_local_blks_written
- Temp and Local blocks read → temp_local_blks_read
- Invalidation messages sent → inval_msgs_total
- Cache resets → cache_resets (для некоторых БД может быть null, оставить null)

Ответ оформи строго в виде маркдаун-таблицы с выравниванием по правому краю для числовых значений, без дополнительных пояснений. Названия баз данных возьми из поля objname.
